Usage¶

Tool for publishing Github issues.

usage: gisrep [-h] {init,report,list} ...

commands¶

command Possible choices: init, report, list

Sub-commands:¶

init¶

Creates a ‘.gisreprc’ configuration file to store Github username and adds the password to the system password manager.

gisrep init [-h] [-f] [-l LOCAL]

Named Arguments¶

-f, --force

Overwrite an existing config file

Default: False

-l, --local Path to a directory in which to save the file

report¶

Publishes a report of nicely formatted Github issues specified by a Github issues search query (see help.github.com/articles/searching-issues-and-pull-requests/)

gisrep report [-h] [-i INTERNAL | -e EXTERNAL] [-c CONFIG] [-p PASSWORD]
              [-u USERNAME]
              query

Positional Arguments¶

query Issues search query

Named Arguments¶

-i, --internal

Tag of internal template to format issues with

Default: “simple_report.md”

-e, --external Path to external template to format issues with
-c, --config Path to gisrep config file
-p, --password Github password
-u, --username Github usename

list¶

Lists internal templates shipped with gisrep that may be used with the ‘report’ command to format the results.

gisrep list [-h]
